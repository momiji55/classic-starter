
# CSS レスポンシブ（ブレークポイント）ルール

classic-starter における、画面幅に応じたレイアウト切り替え（レスポンシブ）の方針を定めたルールです。
このテーマを土台に新規案件を始める際は、本ルールに沿ってメディアクエリを組みます。

## 基本方針

| 項目 | 決定 |
|---|---|
| 設計手法 | **モバイルファースト**（狭い画面を基本に、広い画面を上書きで足す） |
| 方向 | 原則 **min-width 相当**（`width >= ...`）。例外的に下方向の打ち消しが必要なときのみ max-width 相当（`width < ...`） |
| 単位 | **`px`**（ブレークポイントの境界はデバイス幅基準のため px で統一） |
| 値 | **標準ブレークポイントの3段階**から選ぶ。スケール外の値は新規に使わない |

### なぜこの方針か

- **モバイルファースト** … 基本スタイルが軽量な狭幅向けになり、広幅の装飾は加算で済む。記述が「狭→広」と一方向に流れて破綻しにくい。
- **3段階に統一** … 端末は「スマホ / タブレット / デスクトップ」の3区分で十分カバーできる。境界が増えるほど検証コストが上がるため、ボイラープレートでは絞る。
- **px 直書き** … 本テーマはビルド工程（PostCSS / Sass）を持たず、**メディアクエリの条件部では CSS 変数（`var()`）が使えない**。値を直書きし、ドキュメントと `:root` の参考コメントで一元管理する。

## 標準ブレークポイント

| 名前 | 値 | 目安の端末 | 主な用途 |
|---|---|---|---|
| `sm` | **480px** | 小型端末 → 大型スマホ | 左右パディングの拡大など細かな調整 |
| `md` | **768px** | タブレット | 1カラム → 多カラム、要素の横並びなど構造変化 |
| `lg` | **1024px** | デスクトップ | コンテナ余白の拡大、広幅レイアウト |

> メディアクエリの条件部では `var()` を使えないため、上記の値を直書きする。
> 参照用に `common.css` 先頭の `:root` にも同じ値をコメントで記載している。
> 案件ごとに境界を変えたい場合は、本ドキュメント・`:root` コメント・各 CSS の `@media` 値をまとめて更新する。

## 記法

```css
/* 基本（モバイル）スタイルをまず書く */
.inner {
  padding: 0 20px;
}

/* sm 以上で上書き */
@media screen and (width >= 480px) {
  .inner {
    padding: 0 32px;
  }
}

/* lg 以上で上書き */
@media screen and (width >= 1024px) {
  .inner {
    padding: 0 48px;
  }
}
```

- 範囲条件構文（`width >= 480px`）を使う。`min-width: 480px` と等価で、本テーマでは新しい構文に統一する。
- メディアクエリは各 CSS ファイルの末尾「Responsive」セクションにまとめる。セクション見出しに `docs/css-responsive.md 参照` を併記する。

## 適用ルール

### 1. モバイルファースト（min-width 相当）を基本にする

```css
/* OK: 狭い画面を基本に、広い画面で上書き */
.error-404-title { font-size: var(--text-2xl); }

@media screen and (width >= 768px) {
  .error-404-title { font-size: var(--text-3xl); }
}
```

### 2. 下方向の打ち消しが必要なときだけ max-width 相当を使う

横並びを狭い画面で縦積みに戻すなど、「広い画面の状態を狭い画面で打ち消す」場合に限り `width < ...` を使ってよい。

```css
/* prev/next ナビ: md 未満で縦積みに戻す */
@media screen and (width < 768px) {
  .post-entry-nav { flex-direction: column; }
}
```

### 3. 余白・文字サイズはトークンを使う

メディアクエリ内でも、余白は `--space-*`、文字サイズは `--text-*`、色は `--color-*` を `var()` で参照する（境界値の px 直書きはブレークポイントのみに限定する）。

## やってはいけないこと

- 標準スケール外のブレークポイント（`640px`, `600px`, `900px` など）を新規に使う。
- 同じ意味のブレークポイントを `min-width: 768px` と `width >= 768px` で混在させる（新構文に統一）。
- ブレークポイント値を `:root` の CSS 変数で定義して条件部で参照しようとする（**条件部では `var()` が効かない**ため動かない）。

## 適用範囲

本ルールは**画面幅に応じたレイアウト切り替え**を対象とする。
将来 PostCSS 等のビルド工程を導入した場合は、`@custom-media --md (width >= 768px);` のようなカスタムメディアでの一元化に移行してよい（その際は本ドキュメントを更新する）。
