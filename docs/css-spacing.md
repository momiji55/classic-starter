
# CSS 余白（上下スペーシング）ルール

classic-starter における、要素の上下方向の余白の取り方を定めたルールです。
このテーマを土台に新規案件を始める際は、本ルールに沿って余白を組みます。

## 基本方針

| 項目 | 決定 |
|---|---|
| 方向 | **下方向に統一**（`margin-bottom` のみを使う） |
| 単位 | **`rem`**（ルートフォントサイズ基準） |
| 値の刻み | **8px ベースのスケール**を CSS 変数で定義し、`var()` で参照 |

### なぜこの方針か

- **下方向統一** … ブラウザ・WordPress 出力の既定が `margin-bottom` 中心。流れに逆らわず破綻しにくい。「上から積み上げる」感覚に合う。
- **rem** … ユーザーのブラウザ文字サイズ設定に余白も追従するため、アクセシビリティで有利。多案件で再利用するボイラープレートとして汎用性が高い。
- **8px スケール** … 8 は約数が多く半分・倍が作りやすい。デザインツール・UI フレームワークの事実上の標準。

## スペーシングスケール

`:root` に以下の CSS 変数を定義し、余白は必ずこの中から選ぶ。
（`common.css` の先頭で定義する）

```css
:root {
  --space-1: 0.25rem; /*  4px */
  --space-2: 0.5rem;  /*  8px */
  --space-3: 0.75rem; /* 12px */
  --space-4: 1rem;    /* 16px */
  --space-6: 1.5rem;  /* 24px */
  --space-8: 2rem;    /* 32px */
  --space-10: 2.5rem; /* 40px */
  --space-12: 3rem;   /* 48px */
  --space-16: 4rem;   /* 64px */
}
```

- 変数名の数字は **8px を 1 単位とした倍率ではなく、`0.25rem`（4px）刻みの番号**。`--space-6` = `6 × 4px` = 24px と読む。
- 案件ごとに余白トーンを変えたい場合は、`:root` の数値だけを調整すれば全体に一括で反映される。

### 用途の目安

| 変数 | px | 主な用途 |
|---|---|---|
| `--space-2` | 8px | ラベルと値の間など細かい余白 |
| `--space-3` | 12px | 小要素間 |
| `--space-4` | 16px | 標準の要素間 |
| `--space-6` | 24px | 本文ブロック間 |
| `--space-8` | 32px | セクション内の区切り |
| `--space-12` | 48px | セクション間 |
| `--space-16` | 64px | 大きな余白 |

## 適用ルール

### 1. 余白は `margin-bottom` で付ける

```css
/* OK */
.post-item { margin-bottom: var(--space-6); }

/* NG: 上方向は使わない */
.post-item { margin-top: var(--space-6); }
```

### 2. 親要素の末尾は打ち消す

下方向統一では最後の子要素の余白が親からはみ出るため、ラッパー要素では末尾を 0 にする。

```css
.post-item-content > *:last-child {
  margin-bottom: 0;
}
```

### 3. 見出しも下方向のみで表現する

「見出しの前を広げたい」場合でも `margin-top` は使わず、直前要素の `margin-bottom` で間隔を作る。
見出し自体は下に小さめの余白を持たせ、後続の本文と近づける。

```css
/* 見出し: 下に小さめ */
.entry-content :is(h2, h3, h4) {
  margin-bottom: var(--space-3);
}
```

### 4. 本文テキスト間の余白

`common.css` の本文要素（`p`, `ul`, `ol`, `blockquote`, `pre`, `table` 等）は
スケール内の `--space-4`（16px）を基準にする。

```css
.post-item-content :is(p, ul, ol, blockquote, pre, table) {
  margin-bottom: var(--space-4);
}
```

## 整合性に関する注意（body のフォントサイズ）

`rem` の基準は `<html>`（ルート）であり、`html` に `font-size` を指定していなければ
ブラウザ既定の 16px が `1rem` になる。

一方で本文の文字サイズを `body { font-size: 16px; }` と **px で固定すると、本文だけが
ユーザーの文字サイズ設定を無視する**ことになる。余白を rem で組むなら整合させるため、
本文も相対指定にするのが望ましい。

```css
body {
  font-size: 1rem; /* px 固定をやめてユーザー設定に追従させる */
}
```

## やってはいけないこと

- `margin-top` で要素間の余白を作る（方向が混在する）
- スケール外の値（`13px`, `22px`, `30px` など）を直接書く
- 余白に `px` を新規で使う（既存の `px` も順次 `rem` / 変数へ置き換える）

## 適用範囲

本ルールは **要素の上下方向の余白（縦のリズム）** を対象とする。
左右の余白・コンポーネント内部の `padding`・`gap` は別途、同じスケール（`--space-*`）の
利用を推奨するが、本ルールの強制対象ではない。
