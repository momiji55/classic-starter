
# CSS カラートークン ルール

classic-starter における、色の扱い方を定めたルールです。
このテーマを土台に新規案件を始める際は、本ルールに沿って色を組みます。

## 基本方針

| 項目 | 決定 |
|---|---|
| 定義場所 | `common.css` 先頭の `:root` に CSS 変数（`--color-*`）で定義 |
| 参照方法 | 各セレクタでは **必ず `var(--color-*)` で参照**する |
| 直書き | **生の HEX / `rgb()` をルール内に直接書かない** |

### なぜこの方針か

- **案件ごとの色替えが一箇所で済む** … `:root` の値だけ変えれば全ページに反映される。ボイラープレートとして再利用する前提に最適。
- **表記ゆれ・色の散乱を防ぐ** … 似た色が複数できてしまう事故を防ぎ、デザインの一貫性を保つ。
- **ダークモード等の拡張が容易** … 将来 `prefers-color-scheme` で `:root` の値を切り替えるだけで対応できる。

## カラートークン

`:root` に以下の CSS 変数を定義し、色は必ずこの中から選ぶ。
（`common.css` の先頭、スペーシングスケールと同じ `:root` 内で定義する）

```css
:root {
  /* カラー（テキスト） */
  --color-text: #333;          /* 本文 */
  --color-heading: #111827;    /* 見出し・強調テキスト */
  --color-body-soft: #374151;  /* 引用・コードブロックの本文 */
  --color-muted: #4b5563;      /* 補足テキスト */
  --color-caption: #6b7280;    /* キャプション */
  --color-link: #1a73e8;       /* リンク */

  /* カラー（背景） */
  --color-bg: #fff;            /* 標準背景・カード */
  --color-bg-subtle: #f9fafb;  /* 薄い背景 */
  --color-bg-muted: #f3f4f6;   /* やや濃い薄背景（コード・テーブル） */
  --color-dark: #333;          /* 暗い背景（フッター・ボタン） */
  --color-on-dark: #fff;       /* 暗背景の上に乗せるテキスト */

  /* カラー（ボーダー） */
  --color-border: #e5e7eb;        /* 標準ボーダー */
  --color-border-strong: #ccc;    /* 強めのボーダー（入力・ヘッダー） */
  --color-border-muted: #d1d5db;  /* 引用の縦ライン */
}
```

### 用途の目安

| 変数 | 主な用途 |
|---|---|
| `--color-text` | `body` の本文色 |
| `--color-heading` | 見出し、記事タイトル、コード/テーブルの文字 |
| `--color-body-soft` | 引用・コードブロックの本文 |
| `--color-muted` | サイト説明文、メタ情報、補足テキスト |
| `--color-caption` | 画像キャプション |
| `--color-link` | リンク、リンクホバー時の色 |
| `--color-bg` | ページ背景、カード、入力欄、テーブルセル |
| `--color-bg-subtle` | 引用・コードブロック・テーブル見出しの薄い背景 |
| `--color-bg-muted` | インラインコード・テーブルのやや濃い薄背景 |
| `--color-dark` | フッター、ボタン、ホバー時の暗い背景 |
| `--color-on-dark` | 暗背景の上のテキスト（白文字） |
| `--color-border` | カード・テーブル・区切り線の標準ボーダー |
| `--color-border-strong` | 入力欄・ヘッダー下線などのやや強いボーダー |
| `--color-border-muted` | 引用ブロックの左縦ライン |

> `--color-text` と `--color-dark`（共に `#333`）、`--color-bg` と `--color-on-dark`（共に `#fff`）は
> **現状は同じ値だが意味（役割）が異なる**ため別トークンにしている。
> 将来どちらか一方だけ調整したくなった時に分離しておくと安全。

## 適用ルール

### 1. 色は必ず変数で参照する

```css
/* OK */
.post-item { background: var(--color-bg); border: 1px solid var(--color-border); }

/* NG: 生の HEX をルール内に直書きしない */
.post-item { background: #fff; border: 1px solid #e5e7eb; }
```

### 2. 役割に合ったトークンを選ぶ

同じ `#333` でも、文字色なら `--color-text`、暗い背景なら `--color-dark` を使う。
**見た目の色ではなく用途（役割）で選ぶ**こと。

### 3. 案件ごとの色替えは `:root` だけ編集する

ブランドカラーを変える際は各セレクタを触らず、`:root` のトークン値のみ変更する。

## やってはいけないこと

- ルール内に生の HEX / `rgb()` / 色名を直接書く
- スケール外の似た色（例: `#e5e7ea` のような微妙に違う色）を新規追加する
- 用途と異なるトークンを流用する（背景色トークンを文字色に使う等）

## 関連

- 影・トランジション・コンテナ幅も同じ `:root` に `--shadow` / `--transition` / `--container-max` として定義済み。色と同様に直書きせず変数を参照する。
- 余白のルールは [`css-spacing.md`](css-spacing.md) を参照。
